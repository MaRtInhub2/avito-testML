Сначала я разобрал структуру данных и отделил целевую переменную `item_contact` от признаков.  
Отдельно сохранил идентификаторы `query_id` и `item_id`, так как они нужны для финального файла, но не должны участвовать в обучении модели.

Далее я определил категориальные признаки по типу данных (`object` / `category`).  
Для этой задачи я выбрал CatBoost, потому что он:
- нативно работает с категориальными признаками
- устойчив к пропускам
- хорошо подходит для табличных данных

Валидация
Я разбил обучающую выборку на train / validation в пропорции 80/20 со стратификацией по таргету, чтобы сохранить баланс классов.  
Модель обучалась с контролем качества на валидации, используя ROC-AUC как основную метрику.
CatBoost автоматически применяет early stopping — модель останавливается, когда качество на validation перестаёт улучшаться. Это позволяет избежать переобучения и подобрать оптимальное число итераций.

После подбора оптимального количества итераций я переобучил модель на всей обучающей выборке, чтобы использовать максимум доступных данных перед финальным предсказанием на тесте.
